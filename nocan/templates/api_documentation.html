Supabase API Documentation

Note: Make sure to set your Supabase URL and API key in environment variables:
  • SUPABASE_URL - Your Supabase project URL
  • SUPABASE_ANON_KEY - Your Supabase anonymous key

Base URL

All API endpoints are prefixed with: /api/supabase/

Authentication

POST /api/supabase/auth/register/

Register a new user account

{
    "email": "user@example.com",
    "password": "securepassword"
}

POST /api/supabase/auth/login/

Authenticate an existing user

{
    "email": "user@example.com",
    "password": "securepassword"
}

Database Operations

GET /api/supabase/test-connection/

Test the connection to Supabase

Response: 
{
    "success": true,
    "message": "Supabase connection successful",
    "data": {"connected": true}
}

GET /api/supabase/get-data/

Retrieve data from a Supabase table

Query Parameters:
  • table (required) - Table name
  • limit (optional) - Number of records to return (default: 100)
  • offset (optional) - Number of records to skip (default: 0)
Example: /api/supabase/get-data/?table=users&limit=10&offset=0

POST /api/supabase/insert-data/

Insert new data into a Supabase table

{
    "table": "users",
    "data": {
        "name": "John Doe",
        "email": "john@example.com",
        "age": 30
    }
}

PUT /api/supabase/update-data/

Update existing data in a Supabase table

{
    "table": "users",
    "data": {
        "age": 31
    },
    "conditions": {
        "id": 1
    }
}

DELETE /api/supabase/delete-data/

Delete data from a Supabase table

{
    "table": "users",
    "conditions": {
        "id": 1
    }
}

POST /api/supabase/execute-query/

Execute a custom SQL query

{
    "query": "SELECT * FROM users WHERE age > 25"
}

GET /api/supabase/table-info/

Get information about tables in the database

Response Format

All API endpoints return responses in the following format:

{
    "success": true/false,
    "message": "Description of the operation result",
    "data": {
        // Response data or null if error
    }
}

Error Handling

Status Code Description
200 Success
400 Bad Request - Invalid parameters or missing required fields
401 Unauthorized - Authentication failed
500 Internal Server Error - Database or server error

Example Usage with JavaScript

// Test connection
fetch('/api/supabase/test-connection/')
    .then(response => response.json())
    .then(data => console.log(data));

// Get data from a table
fetch('/api/supabase/get-data/?table=users&limit=10')
    .then(response => response.json())
    .then(data => console.log(data));

// Insert new data
fetch('/api/supabase/insert-data/', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
    },
    body: JSON.stringify({
        table: 'users',
        data: {
            name: 'Jane Doe',
            email: 'jane@example.com'
        }
    })
})
.then(response => response.json())
.then(data => console.log(data));

Security Considerations

  • Always use HTTPS in production
  • Store sensitive credentials in environment variables
  • Implement proper authentication and authorization
  • Validate and sanitize all input data
  • Use parameterized queries to prevent SQL injection
